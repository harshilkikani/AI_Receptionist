# PRE-PRODUCTION NOTES

## Pre-record

1. **Three caller clips** — Caleb (easygoing ~32s), Jordan (brisk ~18s),
   Emily (casual ~14s). Phone-filter EQ (bandpass 300–3400 Hz, light
   compression). Use three distinct voices — buyers should not all sound
   like the same person.

2. **Three receptionist replies** — record from the live V11.0 demo
   directly. Set the industry switcher to **Lawrence Realty**, pick
   each persona in the chat shell, run the conversation, capture the
   actual rendered audio. **Use the real voice — Polly Joanna or the
   tenant-configured voice. Do not fake with a different TTS.**

3. **Owner's-phone SMS mocks** — three mocks of Lauren's lock screen
   receiving the three structured texts. Take a real screen recording
   on a real iPhone for the lockbox emergency mock — the lock-screen
   vibration when the SMS arrives is the money shot. The other two
   can be static phone-frame composites.

4. **Screen recordings** — operator portal at `/client/<tenant>?t=<token>`
   set to the real-estate tenant. Today view (sparklines + Worth a
   follow-up). Conversation expansion view (Emily's unified timeline).
   1080p, 60fps. Cursor visible but small.

## Personas — V11.0 registry

All three callers ship as seeded personas in `src/demo_personas.py`:

Their conversation turns in the registry are already production-grade —
use them verbatim for the caller audio scripts. Don't paraphrase.

## Shoot

- Two-person on-camera for hook + close only. Voiceover the middle.
- Separate voiceover tracks, treated room.
- 1-second silent tails after each line for clean cuts.

## Post

- Hard cuts between sections, no transitions.
- Full captions — most hero embeds autoplay muted.
- Export two cuts: captioned-muted (homepage hero) and full-audio
  (clicked-to-play modal).
- Hold the lockbox-emergency SMS frame for an extra beat (the
  `✓ Read` indicator appearance is part of the visual).

---

# TONE

- If "seamless", "robust", "leverage", "cutting-edge", "next-generation"
  slips in → retake.
- Don't say "AI" more than once across the whole video. Say "receptionist"
  or "the system."
- Don't quote market value or comparable sales. The agent's job, not
  the receptionist's — this is encoded in the V11.0 real-estate prompt
  and the script must mirror it.
- Don't promise what the platform doesn't do. No "automated CMAs," no
  "lead nurturing campaigns." Inbound + SMS + showing capture is the
  scope.

# WHAT THIS SCRIPT IS NOT

- Not a demo-day pitch deck script. There is no slide about technology,
  no architecture diagram, no model-name mention.
- Not an objection-handling script. That belongs in
  `LIVE_DEMO_TWO_PRESENTER.md` for in-person meetings where the agent
  pushes back. The video has 3:30 and one job: prove the product is
  real.
- Not a generic AI-assistant ad. Every beat is real-estate-native —
  the vocabulary, the urgency model, the unified timeline. If an HVAC
  agent watches this, they should think "I want the one for me," not
  "this is for me." The HVAC version is the septic SHOWCASE_SCRIPT.md
  (or its V11.0 equivalent, future).
